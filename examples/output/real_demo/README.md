# Real end-to-end pipeline run: notes and results

Generated by [`examples/gpcr_pipeline_real_demo.py`](../../gpcr_pipeline_real_demo.py).
This is a committed record of one actual run (not synthetic/toy numbers) --
`run.log` is the full, unedited console output.

## Why this isn't a GPR68 result

The goal was to test the full 4-tool pipeline on GPR68. That wasn't possible
in this environment: outbound network access to RCSB, the AlphaFold
structure database, and GPCRdb are all blocked at the proxy level in this
sandbox (confirmed via `connect_rejected 403` for all three hosts), and a
prior session's GPCRdb-downloaded `GPR68_active_core.pdb`/`GPR68_inactive_core.pdb`
(referenced in `linkage_pka/gpr68_ph_scan_run/run_gpr68_ph_scan.py`) live in
that session's now-gone scratchpad, not in this repo. Per the user's choice,
this run instead uses the mu-opioid receptor crystal structure (PDB 4DKL,
identified in `README_wsme_gpcr.md`) already bundled in the repo as
`examples/data/gpcr_landscapes_reference/gpcr9i.pdb`, as a stand-in class-A
GPCR -- so the *pipeline plumbing*, as opposed to any GPR68-specific biology,
could be exercised with real tools and real physics rather than fabricated
numbers. Re-running against a real GPR68 structure only requires swapping
`REFERENCE_PDB` in the script.

## What's real vs. substituted in this run

| Tool | Status | Detail |
|---|---|---|
| 1. Protonation (pKa + pH) | **Real** | `wsme_gpcr.pka_predictor.predict_pka_propka` runs PROPKA3 on the actual structure -- structure-aware pKa, not free-solution defaults. |
| 2. AlphaFold/ColabFold ensemble | **Substituted** | GPU + MSA-server access aren't available here (`gpcr_ensemble`'s own README: "not covered by tests since it requires GPU compute and cannot run in this environment"). Replaced with a 5-conformer Calpha-ANM normal-mode ensemble around the reference structure -- a legitimate but much cruder sampling method, disclosed rather than passed off as AlphaFold output. Tool 2's *real* code (`gpcr_ensemble.activation_state.tm3_tm6_distance`/`classify_model`) is still used to score every conformer. |
| 3. Gibbs free energy | **Real** | `gibbs/gpcr_gibbs_energy.py`'s actual AMBER ff14SB + GBn2 implicit solvent + RRHO entropy pipeline (OpenMM + PDBFixer), full minimization, per conformer per pH. |
| 4. Landscape (this branch) | **Real**, unmodified | `gpcr_energy_landscapes` KDE landscape estimator on the real Gibbs energies above. |

## Results

**Tool 1 -- PROPKA3 pKa shifts** (`summary.json`'s `pka_sample`, full list is
41 titratable groups): several buried acidic residues shift several pH units
from their free-solution baseline, e.g. **Asp50: pKa 3.9 -> 7.08** (32%
protonated at pH 7.4, 92% at pH 6.0) and **Asp83: pKa 3.9 -> 6.50**. This is
the same qualitative mechanism (desolvated acidic clusters titrating near
physiological pH) proposed for real proton-sensing GPCRs like GPR68 -- just
demonstrated here on a different receptor.

**Tool 3 -- real Gibbs energies, pH 7.4 vs. pH 6.0** (`gibbs_vs_ph.png`,
`gibbs_kcal_mol_by_ph` in `summary.json`): 10 real OpenMM minimizations (5
conformers x 2 pH values, ~50-90s each). G shifts by **+56 to +66 kcal/mol
(mean +59) going from pH 7.4 to pH 6.0** for every one of the 5 conformers,
consistently -- a real, physically driven destabilization from protonating
the buried acidic residues tool 1 flagged.

**Tool 4 -- landscapes** (`landscape_1d_ph7.4.png`, `landscape_1d_ph6.0.png`,
`landscape_1d_ph_overlay.png`, `merged_ph*.csv`): KDE free-energy landscape
along an auto-detected "DRY-Arg <-> ionic-lock-partner" distance.

## Known limitations of *this specific run* (not of the tools)

1. **N=5 conformers is too small to resolve landscape shape.** The 1D KDE
   plots are close to what any 5-point KDE looks like regardless of the
   underlying physics -- not a converged landscape. A real run should use
   tens to hundreds of conformers (which is what an actual GPU-folded
   ColabFold ensemble, or a longer/denser ANM/MD sampling run, would give
   tool 4 to work with).
2. **The two pH conditions' landscape *shapes* are identical by
   construction.** The collective variables (ionic-lock-like distance,
   NPxxY-DRY distance) were computed on the pre-protonation ANM structure,
   which doesn't depend on pH -- protonation only happens later, inside
   `gibbs`'s own structure prep. So the pH signal is entirely in the
   absolute G values (`gibbs_vs_ph.png`), not in where the landscape sits
   along the x-axis. A tool 2 that samples pH-dependent conformational
   response (e.g. templating on protonated vs. deprotonated structures)
   would be needed to see pH move the landscape's x-axis, not just its
   height.
3. This receptor (mu-opioid, 4DKL) is not itself known to be pH-sensing;
   the pKa/Gibbs machinery demonstrated here is receptor-agnostic and would
   run identically on a real GPR68 structure.

## Reproducing

```bash
pip install -r requirements.txt
SETUPTOOLS_USE_DISTUTILS=stdlib pip install openmm pdbfixer propka  # tool 3 + tool 1 deps
PYTHONPATH=.:gibbs python examples/gpcr_pipeline_real_demo.py
```

Takes ~15 minutes (dominated by 10 real OpenMM minimizations at
`--minimize-iterations 150`).
