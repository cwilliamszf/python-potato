# Real GPR68 pipeline run: notes and results (two-basin sampling)

Generated by [`examples/gpcr_pipeline_gpr68_demo.py`](../../gpcr_pipeline_gpr68_demo.py),
run on **both** real GPR68 (OGR1) homology models the user provided
(`examples/data/gpr68_structures/{active,inactive}/*.pdb`, GPCRdb AFMS).
`run.log` is the full, unedited console output. Per the user's request,
this samples 10 ANM conformers around *each* reference state (20 total)
instead of just the active state, to get broader conformational coverage
without tool 2 (GPU-bound ColabFold folding isn't available here).

## What's real vs. substituted in this run

| Tool | Status | Detail |
|---|---|---|
| 1. Protonation (pKa + pH) | **Real** | `wsme_gpcr.pka_predictor.predict_pka_propka` runs PROPKA3 on the real GPR68 active-state structure. |
| 2. AlphaFold/ColabFold ensemble | **Substituted** | Two 10-conformer Calpha-ANM normal-mode ensembles, one seeded from each real GPR68 reference state, merged into one 20-conformer set. Tool 2's *real* code (`gpcr_ensemble.activation_state`) still scores every conformer. |
| 3. Gibbs free energy | **Real** | `gibbs/gpcr_gibbs_energy.py`'s actual AMBER ff14SB + GBn2 + RRHO pipeline -- 40 real OpenMM minimizations (20 conformers x 2 pH values). |
| 4. Landscape (this branch) | **Real**, unmodified | `gpcr_energy_landscapes` KDE landscape estimator. |

## Results

### Tool 1 -- PROPKA3 (same as the earlier active-only run; unchanged since
it's computed once, on the active-state structure): 68 titratable groups,
notably Asp282 (DPxxY motif) at pKa 9.07 and Asp67/Glu103/Glu149 titrating
within the pH 7.4-6.0 window. See the earlier run's notes for the full
table (git history / superseded `examples/output/gpr68_demo` content at
commit `2900755`).

### Tool 2-substitute -- did seeding from both states actually broaden sampling?

**Yes, qualitatively.** The auto-detected `ionic_lock_like` CV is now
clearly **bimodal**: active-seeded conformers cluster tightly at
11.94-11.95 Angstrom, inactive-seeded conformers cluster tightly at
11.70-11.71 Angstrom (`merged_ph7.4.csv`) -- a real ~0.25 Angstrom
separation between two resolved basins, instead of the single
near-zero-variance blob from the earlier active-only (N=10) run. This is
a genuine improvement in conformational coverage from seeding both states.

**But within-basin diversity is still essentially zero** (~0.003 Angstrom
spread inside each cluster) -- ANM sampling around a fixed reference
point stays very close to that point; it does not explore the
transition pathway *between* the two basins the way a real ensemble
(GPU-folded ColabFold with MSA subsampling, or MD/metadynamics) would.
Practically, this means the two clusters read as two sharp points on the
landscape (see `landscape_2d_ph7.4.png`), not a continuous surface with a
resolved barrier between them -- there's no data in between to describe
what that barrier looks like. Getting an actual barrier/transition-state
picture still needs tool 2 as originally specified (real GPU folding) or
a sampling method that explicitly targets the space between basins (e.g.
interpolation/string-method-style intermediates, or MD started from both
endpoints).

### Tool 3 -- real Gibbs energies, pH 7.4 vs. pH 6.0 (`gibbs_vs_ph.png`)

- pH 7.4: G in [-13934.4, -13891.0] kcal/mol, mean -13909.5
- pH 6.0: G in [-13721.6, -13531.4] kcal/mol, mean -13685.0
- **active-seeded mean shift: +235.7 kcal/mol**
- **inactive-seeded mean shift: +213.3 kcal/mol**
- **overall mean shift: +224.5 kcal/mol**

Both basins are destabilized by a comparable amount going from pH 7.4 to
6.0 (within noise of each other given N=10 per basin), consistent with the
same buried titratable residues (Asp67, Glu103, Glu149, etc.) driving the
pH response regardless of which conformational state they start from.
One conformer (`active_06`) is a clear outlier at pH 6.0 (G=-13531.4 vs.
its pH-7.4 value of -13911.0, a shift of +379.6 kcal/mol, nearly 2x the
typical shift) -- visible as the one line that diverges from the rest in
`gibbs_vs_ph.png`. This is most likely a case where protonation at low pH
pushed that particular ANM-displaced conformer into a different local
energy minimum during minimization (a known sensitivity of single-structure
minimization-based estimates, flagged in `gibbs/README.md`'s own
limitations section), not necessarily a real biological effect -- treat it
as an outlier, not a resolved sub-state, given N=1 at that point.

### Tool 4 -- landscapes

`landscape_1d_ph{7.4,6.0}.png`, `landscape_2d_ph{7.4,6.0}.png`,
`landscape_1d_ph_overlay.png`, colored/marked by seed state where
applicable. As described above, expect two sharp, well-separated peaks
rather than a smooth multi-basin surface -- that's an honest reflection of
what this sampling method produced, not a plotting artifact.

## Known limitations of *this specific run*

1. **Two basins are resolved, but the space between them is not sampled**
   (see above) -- this is the main remaining gap versus a real tool-2
   ensemble.
2. **N=10 conformers per basin** is still small; the outlier conformer
   (`active_06`) shows how much a single minimization can vary.
3. The auto-detected "ionic lock partner" (Asp224) and DRY-Arg (Arg119)
   are sequence/geometry-detected, not validated against GPCRdb generic
   numbering.
4. Geometric CVs are computed pre-protonation, so they don't reflect any
   protonation-induced structural change themselves -- only the Gibbs
   energy carries the pH signal.

## Reproducing

```bash
pip install -r requirements.txt
SETUPTOOLS_USE_DISTUTILS=stdlib pip install openmm pdbfixer propka
PYTHONPATH=.:gibbs python examples/gpcr_pipeline_gpr68_demo.py
```

Takes ~40-45 minutes (40 real OpenMM minimizations at
`--minimize-iterations 150` on a 365-residue, ~5780-atom system).
