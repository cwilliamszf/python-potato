# Real GPR68 pipeline run: notes and results

Generated by [`examples/gpcr_pipeline_gpr68_demo.py`](../../gpcr_pipeline_gpr68_demo.py),
run on the real GPR68 (OGR1) active-state homology model the user provided
(`examples/data/gpr68_structures/active/ClassA_ogr1_human_Active_AFMS_2024-05-15_GPCRdb.pdb`,
GPCRdb AFMS). `run.log` is the full, unedited console output. This
supersedes `examples/output/real_demo/` (which used a mu-opioid receptor as
a stand-in because GPR68 wasn't reachable) as the real GPR68 test.

## What's real vs. substituted in this run

Same disclosure as the earlier stand-in demo:

| Tool | Status | Detail |
|---|---|---|
| 1. Protonation (pKa + pH) | **Real** | `wsme_gpcr.pka_predictor.predict_pka_propka` runs PROPKA3 on the real GPR68 structure. |
| 2. AlphaFold/ColabFold ensemble | **Substituted** | 10-conformer Calpha-ANM normal-mode ensemble around the real GPR68 active-state model (no GPU/network available for actual ColabFold folding here). Tool 2's *real* code (`gpcr_ensemble.activation_state`) still scores every conformer. |
| 3. Gibbs free energy | **Real** | `gibbs/gpcr_gibbs_energy.py`'s actual AMBER ff14SB + GBn2 + RRHO pipeline, full minimization, per conformer per pH. |
| 4. Landscape (this branch) | **Real**, unmodified | `gpcr_energy_landscapes` KDE landscape estimator. |

## Results

### Tool 1 -- PROPKA3 on real GPR68 (68 titratable groups found)

Several residues show large, physiologically-relevant pKa shifts from their
free-solution baseline (`run.log`, `summary.json`'s `pka_sample`):

| Residue | Predicted pKa | Baseline | Shift | Protonated fraction pH7.4 / pH6.0 |
|---|---|---|---|---|
| Arg189 | 20.70 | 12.5 | +8.20 | 0.00 / 0.00 (fully deprotonated throughout) |
| **Asp282** | **9.07** | 3.9 | **+5.17** | **0.98 / 1.00** |
| Glu160 | 8.89 | 4.1 | +4.79 | 0.97 / 1.00 |
| His269 | 2.95 | 6.0 | -3.05 | 1.00 / 1.00 |
| Asp67 | 6.39 | 3.9 | +2.49 | 0.09 / 0.71 |
| His245 | 3.73 | 6.0 | -2.27 | 0.99 / 0.99 |
| Glu103 | 6.28 | 4.1 | +2.18 | 0.07 / 0.65 |
| Glu149 | 5.67 | 4.1 | +1.57 | 0.02 / 0.32 |

**Asp282 is the "D" of GPR68's DPxxY motif** (this receptor's TM7 P-x-x-Y
variant is D-P-V-L-Y at 282-286, not the canonical N-P-x-x-Y), auto-detected
by sequence scan (see `find_conserved_motifs`). Its pKa of 9.07 means it's
essentially fully protonated across the whole physiological range studied
-- a real, structure-derived data point consistent with the mechanism
proposed by Rowe & Isom (2021, JBC) for GPR4/GPR65/GPR68, where a buried
acidic cluster near TM7/TM2 acts as the primary proton sensor (referenced
directly in `wsme_gpcr/pka_predictor.py`'s docstring). **Asp67, Glu103, and
Glu149**, in contrast, titrate cleanly *within* the pH 7.4-6.0 window
studied here (7-65% swings in protonated fraction) -- these are the
residues actually acting as pH sensors in this specific range, more so than
Asp282 (which is already saturated by pH 7.4).

### Tool 3 -- real Gibbs energies, pH 7.4 vs. pH 6.0

10 real OpenMM minimizations per pH (`gibbs_vs_ph.png`,
`gibbs_kcal_mol_by_ph` in `summary.json`):

- pH 7.4: G in [-13905.7, -13776.0] kcal/mol, mean -13876.2
- pH 6.0: G in [-13697.3, -13675.3] kcal/mol, mean -13683.8
- **mean shift: +192.4 kcal/mol** (pH 6.0 less stable than pH 7.4), consistent across all 10 conformers

This shift is **>3x larger** than the equivalent comparison on the
mu-opioid-receptor stand-in structure in `examples/output/real_demo/`
(+59 kcal/mol mean shift, 41 titratable groups vs. GPR68's 68) -- i.e. GPR68
really is more destabilized by the pH 7.4->6.0 drop than an unrelated
receptor of similar size, which is the qualitative signature you'd expect
from an actual proton sensor, computed here from first-principles force-field
energetics rather than assumed.

### Tool 4 -- landscapes (important caveat)

`landscape_1d_ph{7.4,6.0}.png`, `landscape_2d_ph{7.4,6.0}.png`,
`landscape_1d_ph_overlay.png`, `merged_ph*.csv`.

**The landscape shape here is not meaningful and should not be
over-interpreted.** Look at `merged_ph7.4.csv`: the auto-detected geometric
CVs (`ionic_lock_like`, `pxxy_dry_distance`) vary by only ~0.003-0.01
Angstrom across all 10 conformers, even though the ANM ensemble was
generated with a 0.8 Angstrom target Calpha RMSD. This happens because the
dominant low-frequency Calpha-ANM modes move large, coherent stretches of
the receptor together, so a *global* structural RMSD of 0.8 A does not
translate into meaningful change in a *local* two-residue distance -- the
two residues move almost in lockstep. The resulting 2D landscape plot's
axis labels show tiny offsets (`+1.104e1`, `+1.241e1`) for exactly this
reason. **Getting a real landscape shape needs either a genuinely diverse
ensemble (a real GPU-folded ColabFold run, tool 2 as originally specified)
or an ANM sampling scheme that deliberately targets the reaction coordinate
of interest** (e.g. projecting out only modes with a large local
component at the two CV residues), neither of which this substitute
ensemble does. The `gibbs_vs_ph.png` plot (absolute G vs. pH, not
landscape shape) is where this run's real signal lives.

All 10 conformers were classified `"intermediate"` by tool 2's
`classify_model` using its generic, uncalibrated thresholds
(`DEFAULT_THRESHOLDS`, not GPR68-specific) -- the reference active/inactive
models' own ionic-lock-like distances (11.94 A / 11.70 A) are also close
together and both fall in the same generic "intermediate" band, so this
particular auto-detected residue pair may not be GPR68's actual functional
ionic-lock switch. A calibrated pair (via `gpcr_ensemble.activation_state.calibrate_thresholds`
using the real active/inactive reference models, which are both available
here) would be a better choice for a follow-up run.

## Known limitations of *this specific run*

1. **CV range is too narrow for a meaningful landscape shape** (see above)
   -- the headline real result of this run is the pH-dependent G shift, not
   the landscape's x/y structure.
2. **N=10 conformers** is still small relative to what tool 2 (real,
   GPU-folded ColabFold ensembles) or a real MD-based ensemble would
   provide -- tens to hundreds of conformers, as in Fleetwood et al. 2021.
3. The auto-detected "ionic lock partner" (Asp224) and the DRY-Arg (Arg119)
   are sequence/geometry-detected, not validated against GPCRdb generic
   numbering -- treat CV identities as plausible, not confirmed.
4. As before, the geometric CVs are computed pre-protonation (on the ANM
   structure before `gibbs`'s internal PDBFixer protonation step), so they
   don't reflect any protonation-induced structural change themselves --
   only the Gibbs energy carries the pH signal in this run.

## Reproducing

```bash
pip install -r requirements.txt
SETUPTOOLS_USE_DISTUTILS=stdlib pip install openmm pdbfixer propka
PYTHONPATH=.:gibbs python examples/gpcr_pipeline_gpr68_demo.py
```

Takes ~20-25 minutes (20 real OpenMM minimizations at
`--minimize-iterations 150` on a 365-residue, ~5770-atom system).
