# GPR4: real experimental cryo-EM structures under the corrected symmetric protocol

Generated by [`examples/gpcr_pipeline_gpr4_symmetric_demo.py`](../../gpcr_pipeline_gpr4_symmetric_demo.py),
applying the exact same one-protocol-no-exceptions method validated on
GPR68 and GPR132 (full method: [`examples/output/symmetric_string_demo/README.md`](../symmetric_string_demo/README.md))
to **real, independently-solved experimental cryo-EM structures** for
the first time in this repo, rather than AlphaFold-based homology
models. This is also the first receptor in this project with more than
one real active-state structure available, which enabled a fast,
no-OpenMM real-structure pH-titration diagnostic
([`examples/output/gpr4_ph_titration/`](../gpr4_ph_titration/)) as a
bonus sanity check before the expensive Gibbs run.

**Headline result: GPR4's real inactive/active pair shows the clearest
activation-associated TM6 movement of any receptor in this repo (TM6
cytoplasmic displacement 8.1-8.3 Å, roughly double GPR132's and 7x
GPR68's), and — unlike GPR68 or GPR132 — its barrier height is
essentially pH-independent (+0.4%) across the two real pH conditions
tested.** See the important caveat below on why the two GPR4 paths'
*endpoint*-shift comparison is not directly analogous to GPR68/GPR132's,
even though the barrier comparison is reasonably clean.

## Data provenance

11 real cryo-EM structures were uploaded directly by the user (network
access to RCSB/PDBe/AlphaFold DB/GPCRdb is blocked in this environment
-- confirmed via direct `curl` tests, 403 at the proxy). 8 are GPR4 (1
antagonist-bound inactive: 9JFU, with a BRIL fusion in ICL3; 7
active/intermediate at real experimental pH 6.0-8.0, one unspecified);
1 is GPR65 active-only (9BHL); 2 are GPR68 cryo-EM active-only (9BI6,
9BHM), distinct from the earlier AlphaFold-based GPR68 homology models
used elsewhere in this repo. Extraction/cleaning:
[`examples/gpcr_pipeline_cryoem_extract.py`](../../gpcr_pipeline_cryoem_extract.py)
using [`examples/gpcr_pipeline_cryoem_common.py`](../../gpcr_pipeline_cryoem_common.py).

**GPR65 and the two GPR68 cryo-EM structures are active-state only in
this upload -- no inactive counterpart was provided, so no
interpolated-path/Gibbs analysis is possible for them here.** They are
extracted and cleaned (see `examples/data/gpr65_structures/clean_active/`,
`examples/data/gpr68_cryoem_structures/clean_active/`) but otherwise
unused in this report; noted so their absence from the results below
isn't mistaken for an oversight.

### A real data-cleaning bug found and fixed along the way

9JFU carries a BRIL (apocytochrome b562RIL) fusion inserted into ICL3.
The first excision attempt used a fixed 130-residue-by-index window
past the BRIL sequence marker, on the assumption that this
conservatively covered "the whole ~90-106 residue domain plus
flanking linkers." It didn't: the true splice back into native
receptor sequence was only ~99 residues past the marker (verified by
direct sequence comparison against the GPR4 reference), so the fixed
window cut 31 residues too far and silently deleted the first third of
real TM6 with no error raised (5/38 TM6 residues remained). Fixed by
excising only the exact marker match and relying on the existing
alignment step's affine gap penalty to correctly gap around whatever
fusion-domain residues remain, without assuming the fusion's total
length or the marker's position within it. After the fix, 9JFU's
cleaned structure retains 272 residues (up from 240) with 37/38 TM6
residues and 0 identity mismatches against every one of the 7 active
structures.

A second, independent issue was found while identifying TM1-7: DSSP
fragmented GPR4's TM7 at the NPxxY kink into 17/9/8-residue pieces, and
the 9-residue piece containing the *entire* motif never independently
passed the 15-residue minimum-helix-length filter, so the existing
gap-only anchor-merge rule (built for GPR68/GPR132, where the anchor
sits in a gap *between* two already-qualifying runs) found nothing to
merge, and TM7 was silently assigned 12 residues short of the motif it
was supposed to contain. Fixed with a more general merge
(`_merge_runs_across_anchor_span` in
[`examples/gpcr_pipeline_tm_topology.py`](../../gpcr_pipeline_tm_topology.py))
that grows the anchor-containing run outward through short (<=10
residue) gaps until it reaches the minimum length, only if the motif's
full span isn't already contained in one detected run. **Verified no
regression**: GPR68 and GPR132 TM ranges are bit-for-bit identical to
the previously-published values after this change.

Both bugs are the kind the user's original methodology review warned
about ("fail loudly, do not paper over") -- neither raised an
exception; both were caught by explicitly checking that detected
ranges actually cover what they're supposed to, not by assuming the
existing code generalized correctly to new, real data.

## Part 1: fast real-structure pH-titration diagnostic (no OpenMM)

[`examples/gpcr_pipeline_gpr4_ph_titration.py`](../../gpcr_pipeline_gpr4_ph_titration.py)
superposes all 7 real active/intermediate GPR4 structures onto the
invariant core (TM1,2,3,4,5,7, excluding TM6) relative to 8ZCF, and
measures TM6 cytoplasmic displacement -- purely geometric, no Gibbs
scoring, answering "does TM6 position track real experimental pH
among independently-solved active-state structures" before committing
to the expensive run.

| Structure | pH | Invariant-core RMS (Å) | TM6 displacement vs. 8ZCF (Å) | Outlier-reject kept frac. |
|---|---|---|---|---|
| 9LGM | 8.0 | 0.83 | 0.52 | 0.91 |
| 8ZCF (reference) | 7.5 | 0.00 | 0.00 | 1.00 |
| 9JFX | 7.5 | 0.70 | 1.04 | 0.98 |
| 9JFZ (intermediate) | 7.5 | 1.01 | 0.61 | 0.92 |
| 9JFV | 6.8 | 0.65 | 0.73 | 1.00 |
| 8ZCE | 6.0 | 0.40 | 0.26 | 1.00 |
| 9BIP | unspecified | 0.78 | 0.80 | 0.99 |

**Result: no detectable pH-dependent TM6 movement among active-state
structures**, and this is the expected, honest result, not a null
finding to explain away: all 7 structures are active-state, so there's
no activation transition to see here by construction -- only
within-basin structural noise between independent depositions. That
noise floor is itself useful context: three structures solved at the
*identical* nominal pH (7.5: 8ZCF, 9JFX, 9JFZ) show 0/1.04/0.61 Å
spread relative to each other -- comparable in magnitude to the
0.26-0.80 Å spread seen across the *different* pH values in this set.
In other words, structure-to-structure variability at fixed pH is not
smaller than the apparent pH-to-pH variability here; there is no
real signal to separate from that noise in the active-only comparison.
This motivates Part 2 below, which compares against the one real
inactive structure available (9JFU) instead.

Full data: [`summary.json`](../gpr4_ph_titration/summary.json),
[`tm6_displacement_vs_ph.png`](../gpr4_ph_titration/tm6_displacement_vs_ph.png).

## Part 2: symmetric active/inactive pipeline + real Gibbs energies

Two pH-matched real-structure paths, each between the one real
inactive structure (9JFU, antagonist-bound) and a real active
structure solved near the same pH -- chosen specifically to use two
INDEPENDENTLY SOLVED active-state structures at two different real
experimental pH values, rather than reusing one active structure and
only varying scored pH (which is what the GPR68/GPR132 runs did, for
lack of real pH-resolved structures):

- **Path A**: 9JFU (inactive) -> 8ZCF (active, deposited at pH 7.5), scored at pH 7.4.
- **Path B**: 9JFU (inactive) -> 8ZCE (active, deposited at pH 6.0), scored at pH 6.0.

TM1-7 identified once from 8ZCF (all active structures share identical
native numbering, verified during extraction) and reused for both
paths. Superposition and interpolation both had to be adapted for real
structures with genuinely different residue coverage (9JFU has two
small real gaps -- a 1-residue gap at 63, a 6-residue disordered-loop
gap at 211-216 -- that the active structures don't share):
`superpose_invariant_core_common` fits on the intersection of resolved
invariant-core resnums (143/144 usable, 99.3%) instead of requiring
exact matching coverage, and `build_interpolated_path_common` matches
images by resnum instead of by list position.

### Real Gibbs energies along both paths (11 images each, 150-iteration minimization)

| | Path A (pH 7.4) | Path B (pH 6.0) |
|---|---|---|
| G, active endpoint (f=0.0) | -9515.70 kcal/mol | -9278.81 kcal/mol |
| G, inactive endpoint (f=1.0) | -9393.63 kcal/mol | -9269.05 kcal/mol |
| Endpoint shift (inactive - active) | +122.07 kcal/mol | +9.76 kcal/mol |
| Peak G (f=0.5, both paths) | -8913.19 kcal/mol | -8674.11 kcal/mol |
| **Barrier (peak - lower endpoint)** | **602.51 kcal/mol** | **604.70 kcal/mol** |
| Core endpoint CA RMSD post-fit | 3.19 Å | 3.21 Å |

`gibbs_vs_path.png` for each path shows G rising smoothly and roughly
symmetrically from the active endpoint to a peak exactly at the
midpoint (f=0.5) and back down toward the inactive endpoint, the same
qualitative shape seen in every prior interpolated-path run in this
repo. `rmsd_sanity_check.png` confirms RMSD-to-active and
RMSD-to-inactive cross linearly at f=0.5 for both paths, i.e. the
150-iteration relaxation preserved the interpolated path rather than
collapsing into either endpoint basin.

**Barrier height is essentially pH-independent: 602.5 kcal/mol at
pH 7.4 vs. 604.7 kcal/mol at pH 6.0, a +0.4% change** -- both directly
comparable because both paths share the same inactive endpoint (9JFU)
and, in both paths, the active endpoint happens to be the lower one
(so "barrier relative to the lower endpoint" means the same thing in
both cases). This is a genuinely different qualitative result from
both other receptors in this repo: GPR68's corrected barrier dropped
23.7% (60.25 -> 45.99 kcal/mol) from pH 7.4 to 6.0, and GPR132's rose
2.5% (344.54 -> 353.29 kcal/mol). GPR4's barrier essentially doesn't
move.

### Important caveat: the endpoint-shift numbers are NOT directly comparable to GPR68/GPR132's

GPR68 and GPR132's pH comparisons held the active structure fixed and
only varied the *scored* pH -- a clean, single-variable comparison
isolating the pH effect on one fixed conformation. **This GPR4 run
cannot make that comparison because no single active structure has
been solved at both pH values** -- Path A and Path B use two different,
independently-solved active structures (8ZCF vs. 8ZCE) at their own
real deposition pH. The two paths' raw endpoint G values (and the very
different endpoint shifts, +122.1 vs. +9.8 kcal/mol) therefore reflect
some unknown mixture of (a) the real pH-driven protonation effect and
(b) real structural differences between 8ZCF and 8ZCE as independent
cryo-EM depositions (different resolution, different modeled
side-chain conformations, etc.) -- **this run cannot separate the two,
and the endpoint-shift numbers above should not be read as a clean pH
effect the way GPR68's/GPR132's are.** The barrier comparison above is
on firmer ground because both paths share the identical inactive
endpoint, but even there, some of the barrier's shape reflects
real differences between the two active starting geometries, not pH
alone. A true isolated-pH comparison for GPR4 would require scoring
the same active structure (e.g. 8ZCF) at both pH 7.4 and 6.0, exactly
as done for GPR68/GPR132 -- not done here because the point of this
run was specifically to use the real pH-resolved structures GPR4
uniquely offers (see module docstring); a same-structure two-pH GPR4
run would be a natural, cheap follow-up if a cleaner pH-only
comparison is wanted.

| **TM6 cytoplasmic displacement** | **1.18 Å** | **4.87 Å** | **8.34 Å / 8.14 Å** |
| **TM6 gate (>= 4 Å)** | **FAILS** | passes (marginal) | **PASSES clearly** |
| Barrier, pH 7.4 | 60.25 kcal/mol | 344.54 kcal/mol | 602.51 kcal/mol |

| Barrier pH-response | -23.7% (drops) | +2.5% (rises slightly) | **+0.4% (flat)** |
| Endpoint shift (inactive-active), pH 7.4 | +15.56 kcal/mol | +28.48 kcal/mol | +122.07 kcal/mol* |
| Endpoint shift (inactive-active), pH 6.0 | +13.71 kcal/mol | +49.00 kcal/mol | +9.76 kcal/mol* |

\* GPR4's endpoint shifts are not a clean pH-only comparison -- see
caveat above; included for completeness, not for direct comparison to
the other two receptors' cells in that row.

GPR4 is the only receptor whose real active/inactive pair clearly and
unambiguously passes the genuine-activation-pair gate -- consistent
with comparing an experimentally solved antagonist-bound inactive
structure against experimentally solved active structures, rather than
two independently-generated homology models of uncertain relationship
to any real conformational transition (GPR68's case). GPR132's
homology-model pair passes only marginally (4.87 Å, just above the 4 Å
threshold) and its absolute barrier magnitude (344-353 kcal/mol) sits
between GPR68's and GPR4's -- plausibly reflecting that its pair, while
real, may capture a smaller fraction of the true conformational
transition than GPR4's real structures do.

## Sanity gates and caveats (same standards as GPR68/GPR132)

- **TM6 gate**: both GPR4 pairs pass clearly (8.1-8.3 Å, more than
  double the 4 Å threshold) -- this is a genuine activation-associated
  transition, not a frame artifact.
- **N=1 per (image, path)**: no error bars. No difference reported here
  -- including the barrier's near-flatness across pH -- should be
  described as statistically significant.
- **Gate A (uncalibrated absolute energetics) is still open**: PROPKA3
  has not been validated against an experimental buried-carboxylate
  pKa benchmark in this pipeline (see `linkage_pka`'s `FINDINGS.md` for
  a related, sign-inverted PB-calibration attempt). Absolute G
  magnitudes here (order -9000 kcal/mol) are uncalibrated; barrier
  heights and their pH-response are the more defensible comparison,
  and even those should be read as first estimates from a
  linear-interpolation-plus-relaxation path, not a converged string
  method (no reparametrization, no tangent-force projection).
- **Barrier-vs-path-length monotonicity**: not directly checkable
  across receptors here since "path length" (raw endpoint RMSD) isn't
  uniquely defined the same way across differently-covered real
  structures, but within each GPR4 path the barrier sits at the
  interpolation midpoint (f=0.5) in both cases, consistent with the
  smooth, unimodal profile seen in every other run in this repo.
- **Endpoint-shift caveat**: see the dedicated section above --
  GPR4's two paths use different active structures, so their
  endpoint-shift comparison conflates real structural differences with
  the pH effect, unlike GPR68/GPR132's single-active-structure design.

## Reproducing

```bash
pip install -r requirements.txt
SETUPTOOLS_USE_DISTUTILS=stdlib pip install openmm pdbfixer propka
apt-get install -y dssp   # for real DSSP-based TM helix detection

PYTHONPATH=. python examples/gpcr_pipeline_cryoem_extract.py           # extract/clean all 11 structures, ~1 min
PYTHONPATH=. python examples/gpcr_pipeline_gpr4_ph_titration.py        # Part 1, no OpenMM, ~1 min
PYTHONPATH=.:gibbs python examples/gpcr_pipeline_gpr4_symmetric_demo.py  # Part 2, ~30-35 min (22 real Gibbs runs)
```
